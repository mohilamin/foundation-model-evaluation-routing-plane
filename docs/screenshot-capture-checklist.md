# Screenshot Capture Checklist

Screenshots are not required for tests, but they make the repo easier to review quickly. Suggested captures:

- Executive overview
- Architecture diagram
- Key dashboard section
- Scorecards/report output
- API sample response
- Workflow or decision view
- Generated artifact table

## Capture Tips

- Run the pipeline before opening the dashboard.
- Use a clean browser window.
- Capture the section title and at least one visible metric/table.
- Store images under `docs/screenshots/` or `screenshots/` if the repo uses that convention.
