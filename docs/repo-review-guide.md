# Repository Review Guide

## Recruiter Review Path

1. Read the README executive summary and Why I Built This section.
2. Review the generated artifacts listed there.
3. Check `docs/validation-log.md`.
4. Skim recruiter or demo docs if present.

## Hiring Manager Review Path

1. Start with the business problem.
2. Review what the project proves.
3. Look at scorecards and validation results.
4. Check known limitations and production roadmap.

## Senior Engineer Review Path

1. Inspect architecture docs.
2. Inspect `src/` modules.
3. Inspect tests.
4. Review `docs/design-decisions.md`.
5. Review generated scorecards and artifacts.

## Demo Path

1. Run the pipeline command from `docs/validation-log.md`.
2. Launch the dashboard or API if available.
3. Show one generated artifact.
4. Explain one design tradeoff.

## Files To Inspect

- `README.md`
- `docs/design-decisions.md`
- `docs/tradeoffs-and-simplifications.md`
- `docs/validation-log.md`
- `docs/production-roadmap.md`
- `src/`
- `tests/`
