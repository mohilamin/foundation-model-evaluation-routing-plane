# STAR Story

## Situation

Enterprise AI teams use multiple model candidates with different tradeoffs. A single "best model" decision is not enough when requests have different domains, budgets, safety needs, and latency SLAs.

## Task

Build a model routing control plane that evaluates candidate models, routes requests under constraints, explains decisions, and simulates canary/shadow rollout risk.

## Action

Created synthetic evaluation tasks, a simulated model registry, deterministic outputs, evaluation scorecards, Pareto frontiers, routing explanations, canary/shadow reports, monitoring scorecards, FastAPI endpoints, Streamlit dashboard, Docker, CI, and tests.

## Result

Produced a reproducible AI infrastructure portfolio project demonstrating model evaluation, model governance, routing policy, and deployment-ready prototyping.
